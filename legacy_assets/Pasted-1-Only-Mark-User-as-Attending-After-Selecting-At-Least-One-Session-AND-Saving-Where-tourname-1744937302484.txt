1. 🛠 Only Mark User as “Attending” After Selecting At Least One Session AND Saving

Where: tournament_routes.py (POST on /tournament/<id>)

Fix:

Only set attending = True if the user selects at least one Day or Night session and clicks Save.
Sample logic:

selected_sessions = {}
for key in request.form:
    if key.startswith("session_"):
        day = key.split("_", 1)[1]
        selected_sessions.setdefault(day, []).append(request.form[key])

if selected_sessions:
    if not user_tournament:
        user_tournament = UserTournament(user_id=user.id, tournament_id=tournament.id)

    user_tournament.sessions = selected_sessions
    user_tournament.attending = True
    user_tournament.open_to_meet = request.form.get("open_to_meet") == "yes"

    db.session.add(user_tournament)
    db.session.commit()

    return redirect(url_for("main_routes.home"))
else:
    flash("Please select at least one Day or Night session before saving.")
    return redirect(url_for("tournament_routes.tournament_detail", tournament_id=tournament.id))
Also:

In home.html → only list tournaments where:
user_tournament.attending == True and user_tournament.sessions != {}
On the tournament detail page:
❌ Do not display “You’re attending this tournament” if no sessions are selected
